¿TRANSPORTE CON IoT?

El proyecto esta basado en la simulación del comportamiento humano en el transporte público, permitiéndonos concluir que la implementación del transporte con IoT integra tecnología conectada para monitorear vehículos, rutas y tráfico en tiempo real. Permite optimizar la operación, reducir costos y aumentar la seguridad mediante el análisis de datos. Este enfoque impulsa una movilidad más eficiente, sostenible y adaptada a las necesidades modernas del transporte inteligente.

METODOLOGÍA

  • 1. Comprensión del Problema

    El proyecto busca analizar y optimizar el sistema de transporte público una ciudad inteligente con IoT, tomando como modelo el Transmilenio de la ciudad de Bogotá.
    Su objetivo principal es mejorar la eficiencia y experiencia del usuario mediante el análisis de datos sobre rutas, estaciones, usuarios y vehículos, tomando decisiones a partir del conocimiento generado.

  • 2. Comprensión de los datos

    Se trabajó con datos simulados que representan el funcionamiento del sistema Transmilenio. Identificamos entidades como Usuarios, Rutas, Estaciones, EstacionesxRuta y Vehículos.
    Se analizaron las características y relaciones de los datos creados, verificando su coherencia y corrigiendo inconsistencias para asegurar que reflejaran adecuadamente un entorno real de transporte público.

  • 3. Preparación de los datos

    Se realizó la migración y transformación de los datos simulados desde el modelo relacional hacia un Data Warehouse multidimensional.
    Durante el proceso ETL, se limpiaron, seleccionaron e integraron los datos de Usuarios, Rutas, Estaciones, EstacionesxRuta y Vehículos, obteniendo una base consistente y lista para el análisis del sistema de transporte.

  • 4. Modelado

    Con la información cargada en el modelo multidimensional, se desarrollaron modelos analíticos para identificar patrones y comportamientos dentro del sistema de transporte mediante la herramienta Power BI.

  • 5. Evaluación

    En esta fase se evaluaron los resultados obtenidos del modelado para verificar que cumplieran con los objetivos del proyecto.

  • 6. Despliegue

    Finalmente, los resultados se presentaron mediante reportes y paneles de visualización que muestran indicadores clave del sistema de transporte.

ETL

El proceso ETL (Extract, Transform, Load) es una metodología utilizada para extraer datos de diferentes fuentes, transformarlos según las necesidades del análisis y cargarlos en un sistema de almacenamiento central, como un Data Warehouse. En el proyecto del sistema de transporte público, el ETL se aplicó para trasladar la información simulada del modelo relacional al modelo multidimensional, garantizando su limpieza, consistencia y estructura adecuada. Durante este proceso, se extrajeron los datos de las entidades principales (Usuarios, Rutas, Estaciones, EstacionesxRuta y Vehículos), se transformaron mediante la estandarización de formatos y la creación de indicadores, y finalmente se cargaron en el Data Warehouse, permitiendo realizar análisis más eficientes y obtener una visión integral del funcionamiento del sistema.

Dimension Ruta

Esta dimensión carga los datos de las rutas desde una fuente OLE DB hacia su destino en el Data Warehouse. Es un flujo directo, ya que los datos no requieren transformaciones intermedias.

Dimension Vehiculo

Incluye un flujo más complejo: los datos se extraen desde OLE DB, se aplican transformaciones mediante una columna derivada (para crear o ajustar valores) y luego una búsqueda (lookup) para relacionar claves con otras tablas antes de cargarse en el destino OLE DB.

Dimension estacion

Al igual que la dimensión Ruta, este flujo realiza una carga directa desde la fuente OLE DB hacia el destino. No requiere transformaciones adicionales, ya que los datos están listos para insertarse en la tabla dimensional.

Dimension Usuario

El flujo extrae los datos desde OLE DB, genera nuevos campos o formatea datos mediante una columna derivada, y realiza una búsqueda para vincular información con otras tablas antes de cargar los datos en el destino OLE DB.

ANALISIS

Description

Este panel ofrece una visión integral del sistema de transporte, combinando indicadores clave —como el total de vehículos, rutas, eventos y estaciones— con mapas y gráficos interactivos. Permite analizar la distribución y cobertura de la red, identificar zonas de mayor demanda y observar la evolución del volumen de usuarios en diferentes periodos. Además, facilita el seguimiento del comportamiento de los usuarios por hora, día y mes, destacando patrones y picos de uso para optimizar la operación del sistema.

Informacion general

Este panel resume el estado del sistema de transporte mostrando totales de vehículos, rutas, eventos y estaciones, junto con un mapa que ubica las estaciones y permite visualizar la cobertura y distribución de la red en la ciudad.

Informacion recuento usuarios

El panel integra mapas y gráficos interactivos que reflejan el volumen de usuarios por estación y su evolución mensual. Esta representación geográfica permite identificar las zonas con mayor demanda y optimizar la gestión del servicio de transporte en toda la red.

Informacion flujo de personas

Visualización del comportamiento de los usuarios por hora, día y mes, destacando los picos de uso y las tendencias generales del sistema de transporte.
